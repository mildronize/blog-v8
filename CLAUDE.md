## Project Overview

Personal blog (thadaw.com) built with **Zola** (Rust static site generator, v0.19.2) using the **Serene** theme (git submodule at `/themes/serene/`). Interactive frontend components are built with **React 19 + TypeScript + Vite** in the `/snippets/` directory. Backend API runs on **Azure Functions + Hono**. Hosted on **Azure Static Web Apps**.

## Common Commands

All commands assume you're in the repo root unless noted.

```bash
# Development
make dev                    # Full dev setup: install deps, build post artifacts, then zola serve --drafts
make serve-watch            # Just zola serve with drafts enabled

# Building
make build                  # Production build (zola build + post-build steps)
make blog-build             # Zola build only
make post-build             # Resume + ID mapper + clean JS + search index

# Snippets (frontend/build scripts) — run from /snippets/
bun install                 # Install JS dependencies
bun dev                     # Vite dev server + search metadata
bun build                   # Build JS bundles
bun test                    # tsc type-check + bun test
bun run add-id              # Auto-add unique IDs to posts missing them
bun generate-id-mapper      # Generate post ID → slug mapping
bun build:search-metadata   # Build search index files
bun resume:build            # Build resume HTML from resume.json
```

## Architecture

### Two-layer build system

1. **Zola** generates static HTML from Markdown content + Tera templates (`/templates/`)
2. **Vite** (in `/snippets/`) builds React components that hydrate into the static pages

Vite entry points: `404`, `post`, `search-modal`, `search-placeholder`, `commander` — output to `/static/js/`, served at `/js/` in production.

### Content (`/content/`)

- **Posts** in `/content/posts/` — Markdown with TOML front matter (`+++` delimiters)
- Each post has an `[extra].id` field (8-char unique ID) for short URLs (`/s/{id}`)
- IDs are auto-generated by a GitHub Action on push to main
- Other sections: `/content/projects/`, `/content/talks/`, `/content/about/`

### Snippets (`/snippets/`)

The Vite/React/TypeScript project containing:
- `src/components/` — React components (404 resolver, post page features, search UI)
- `src/server/` — Azure Functions backend (Hono app: search API, reactions, health)
- `src/libs/` — Shared libraries (content processing, search, ID generation)
- `src/scripts/` — Build scripts (add-id, build-search-metadata)

### Build artifacts (`/public/`)

- `/public/js/` — Vite output bundles
- `/public/api/` — Generated JSON (id-mapper, search indices, post metadata)
- `/public/resume/` — Generated resume HTML

### Key config files

- `config.toml` — Zola config (base_url, theme, taxonomies, extra settings like Giscus comments and reactions)
- `snippets/vite.config.ts` — Vite multi-entry build config
- `static/staticwebapp.config.json` — Azure SWA routing and headers

## CI/CD

- **Auto Add Post ID** (`.github/workflows/auto-add-id.yml`) — On push/PR to main, auto-adds missing post IDs and creates a merge PR
- **Deploy** (`.github/workflows/deploy-static-web-apps.yml`) — Builds and deploys to Azure SWA; creates preview environments on PRs

## Post Format

```toml
+++
title = "Post Title"
description = "Description"
date = "YYYY-MM-DD"

[taxonomies]
tags = ["tag1", "tag2"]
categories = ["category"]

[extra]
id = "xxxxxxxx"  # auto-generated 8-char ID
unsplashImgCoverId = "xxx"  # optional cover image
+++
```

Content is written in Markdown. Posts can be single `.md` files or directories with `index.md` (for co-located assets like images).
